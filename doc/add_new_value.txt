NEW TEST AREA J:  Dual-Process Partial-Failure Scenarios (Design §4.3.6)
======================================================================

The design explicitly calls out that Agent and Service initialize FIPS independently.
Edge cases where one succeeds and the other fails need dedicated coverage.

Test

---   ----        ---  -----------                                                                     --------

                       for the Service process (e.g., via file lock timing or separate config).        macOS
                       Verify entire client enters disabled state per Strict policy.

102   Functional  P0   Agent FIPS Fails, Service FIPS OK (Strict): Reverse of 101. Verify entire       Win x64 /
                       client enters disabled state.                                                   macOS

103   Functional  P1   Agent FIPS OK, Service FIPS Fails (Permissive): Verify both processes fall      Win x64 /
                       back to Non-FIPS Mode (no partial/mixed compliance state).                      macOS

104   Functional  P1   Agent FIPS Fails, Service FIPS OK (Permissive): Verify both fall back to        Win x64 /
                       Non-FIPS consistently, not a mixed state.                                       macOS

105   Observability P1 Split-Process Log Correlation: Verify logs from Agent and Service can be        Win x64 /
                       correlated (by timestamp and process ID) to reconstruct the unified             macOS
                       FIPS enablement outcome for audit.

======================================================================

macOS supports both Intel (x86_64) and Apple Silicon (arm64). The FIPS provider
may behave differently across architectures.
